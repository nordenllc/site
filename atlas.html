Arlington Commercial Vitality Atlas

Where Arlington's commercial corridors are strong, stressed, and adapting.

v0.1 prototype demo seed data Rosslyn–Ballston corridor

The Atlas is a civic geospatial web app for understanding where Arlington's commercial corridors are strong, stressed, and adapting. It makes the commercial-tax-base challenge spatially legible — not just “office vacancy is a problem,” but which buildings, parcels, corridors, and reuse opportunities matter most.

The v0.1 prototype focuses on the Rosslyn–Ballston corridor and lets users read properties through a set of map modes. It is built as a public-data atlas, not a generic GIS viewer: the goal is a legible picture of a specific civic question, not another blank canvas of layers.

Map modes

Each mode re-reads the same corridor through a different question.

  • commercial stressWhere vacancy, assessment decline, and turnover concentrate.
  • adaptive reuseWhich buildings are the strongest candidates to become something else.
  • street vitalityWhere the ground floor and the street are active — or aren't.
  • assessment changeHow taxable value is moving, parcel by parcel.
  • active projectsWhat is in the development pipeline, and where.

Every property has a profile

The Atlas is organized around places, not layers. Select a property and you get a profile: address, corridor, scores, assessment signals, project status, zoning and reuse notes, a confidence level, and the sources behind each value.

Confidence and sources sit on the same footing as the scores themselves. An atlas of a contested civic question has to be honest about what it knows, how well it knows it, and where the number came from.

From demo data to a reproducible pipeline

The current version uses demo seed data to prove the interface and the product concept. The next serious step is replacing that with a reproducible pipeline built from Arlington public datasets:

  • parcels and assessments,
  • zoning and adaptive-reuse records,
  • development tracking and active project status,
  • transit proximity,
  • and commercial-place identity resolution — deciding which records describe the same real-world property.

That last step is where the Atlas meets Masora. A parcel, a storefront, an assessment record, and a development application often describe the same place under four different identifiers; resolving them to one stable place identity is what lets a score be about a property rather than a row in one file. The Atlas is, in part, a proving ground for that idea on real public data.

Status

This is an early prototype running on demo data, published in the open. The interface and concept are real; the numbers are placeholders until the public-data pipeline lands. We're sharing it now to invite scrutiny, corrections, and conversation — not as a finished product.

Why we build this in the open

The data that describes Arlington's commercial future is largely public already. Putting it together into something legible shouldn't require private access or proprietary tooling. We're publishing the Atlas, and intend to publish the pipeline that feeds it, so the picture can be checked, reused, and argued with — which is how civic data earns trust.
